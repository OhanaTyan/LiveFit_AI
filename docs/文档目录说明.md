详细的文档命名规范请参考：`docs/naming_convention.md`

## 4. 如何使用本文档

1. **项目概述**：阅读 `docs/core/README.md` 和 `docs/core/technical_architecture.md` 了解项目的基本情况和技术架构
2. **开发指南**：阅读 `docs/development/开发环境配置.md` 配置开发环境
3. **功能开发**：根据需要阅读 `docs/features/` 下对应模块的文档
4. **API 集成**：阅读 `docs/api/rest_api.md` 了解 API 设计和使用方法
5. **问题诊断**：遇到问题时查阅 `docs/operations/troubleshooting.md`

## 5. 文档维护

- 文档采用 Markdown 格式编写
- 所有文档应保持最新，与代码同步更新
- 新功能开发时应同步编写或更新对应的文档
- 文档变更应遵循项目的版本控制流程

## 6. 贡献指南

欢迎对项目文档进行贡献！如果您发现文档中的错误或有改进建议，请按照以下流程进行：

1. Fork 本项目
2. 创建功能分支 (`git checkout -b feature/update-documentation`)
3. 提交更改 (`git commit -m 'docs: 更新文档内容'`)
4. 推送到分支 (`git push origin feature/update-documentation`)
5. 提交 Pull Request
